# 智扫通 AI Agent 项目报告与简历写法

## 1. 项目名称

**智扫通——基于 LangChain、LangGraph 与混合 RAG 的扫地机器人智能客服系统**

英文名称可写为：

**Smart Cleaning Robot Customer Service Agent with LangGraph and Hybrid RAG**

## 2. 项目概述

智扫通是一个面向扫地机器人和扫拖一体机器人场景的 AI 智能客服项目。系统通过大语言模型理解用户问题，根据问题类型自主调用知识检索、天气、用户信息和使用记录等工具，为用户提供产品选购、故障排查、维护保养、环境适配和个人使用报告等服务。

项目将通用大语言模型与本地机器人知识资料、用户月度使用数据结合。普通知识问题通过混合 RAG 获取有来源依据的答案；个人使用报告则通过 LangGraph 固定状态流完成数据查询、知识补充和报告生成，降低 Agent 漏步骤或无数据编造的风险。

## 3. 项目解决的问题

传统客服系统通常存在以下问题：

- 只能根据固定关键词匹配答案，难以理解自然语言问题。
- 通用大模型不了解企业内部产品资料，容易产生幻觉。
- 单纯使用向量检索时，对故障码、型号和明确关键词的匹配不够稳定。
- 自由 Agent 在复杂业务流程中可能漏掉必要步骤。
- 知识文件发生修改或删除后，旧向量难以被准确清理。
- 回答没有资料来源，用户无法判断内容是否可靠。

本项目分别通过 Agent 工具调用、混合 RAG、LangGraph 状态流、来源引用和 SQLite Manifest 解决这些问题。

## 4. 主要用途

### 产品知识问答

回答扫地机器人功能、参数、使用方式和适用环境等问题。

示例：

- 小户型应该选择什么类型的扫地机器人？
- 扫拖一体机器人适合木地板吗？
- 扫地机器人为什么需要建立地图？

### 故障排查

结合本地故障资料，为用户提供初步排查步骤。

示例：

- 机器人无法自动回充怎么办？
- 主刷被毛发缠绕应该如何处理？
- 设备反复提示悬崖传感器异常是什么原因？

### 维护保养

根据本地维护资料回答主刷、边刷、滤网、尘盒和电池等部件的维护问题。

示例：

- HEPA 滤网多久更换一次？
- 主刷应该怎样清理？
- 长时间不用机器人时应该如何保存电池？

### 环境适配建议

Agent 可以组合用户位置、天气和机器人知识工具，生成与温度、湿度等环境相关的使用建议。

当前天气数据为演示工具，后续可替换为真实天气 API。

### 个人使用报告

根据用户 ID 和月份查询 CSV 中的使用记录，结合 RAG 检索到的维护知识生成 Markdown 报告。

报告内容包括：

- 用户居住环境和设备使用特征。
- 清扫覆盖率与异常情况。
- 主刷、滤网等耗材状态。
- 与同类型用户的使用效果对比。
- 有知识来源依据的保养建议。

## 5. 系统架构

```text
Streamlit 聊天界面
        ↓
ReactAgent 请求路由
        ├── 普通知识问题
        │       ↓
        │   LangChain Agent
        │       ↓
        │   工具调用 / Hybrid RAG
        │       ├── Chroma 向量检索
        │       ├── BM25 关键词检索
        │       ├── 轻量 Reranker
        │       └── 相关度阈值与来源引用
        │
        └── 使用报告问题
                ↓
            LangGraph 状态流
                ├── 解析用户与月份
                ├── 查询 CSV 使用记录
                ├── 检索维护知识
                └── 生成最终报告

知识文件 → 文档加载与切片 → Gemini Embedding → Chroma
                         ↑
                  SQLite Manifest
               管理新增、修改与删除
```

## 6. 使用的具体技术

### Python

项目主要开发语言。使用面向对象、数据类、类型注解、上下文变量和标准库完成服务封装、配置读取、CSV 解析、日志及测试。

### Streamlit

用于构建聊天式 Web 界面，包括：

- 用户消息输入。
- 对话历史展示。
- Agent 回答流式展示。
- Session State 会话状态管理。

### LangChain

用于构建 ReAct 风格 Agent 和工具调用系统。

具体使用：

- `create_agent` 创建智能体。
- `@tool` 封装 RAG、天气、用户和数据查询能力。
- Middleware 记录工具调用与模型请求。
- PromptTemplate 和输出解析器构建 RAG Chain。
- 动态 Prompt 支持不同业务场景。

### LangGraph

用于定义个人使用报告的确定性工作流。

报告状态流：

```text
resolve_context
    ↓
load_record
    ├── 无记录 → 返回提示并结束
    └── 有记录
          ↓
retrieve_knowledge
          ↓
generate_report
```

相比完全依赖 Agent 自主规划，LangGraph 可以确保报告生成严格执行必要步骤，并明确处理无数据分支。

### RAG

使用检索增强生成技术，将本地知识库内容作为模型回答依据。

流程：

```text
用户问题 → 检索相关分片 → 重排和阈值过滤
        → 将资料注入 Prompt → 模型生成带引用回答
```

### 混合检索

项目组合了两种检索方式：

- Chroma 向量检索：处理语义相似但表达不同的问题。
- BM25 关键词检索：提高型号、故障码和明确产品词的召回能力。

### 轻量 Reranker

将向量相似度、BM25 得分和查询词覆盖率加权计算：

```text
最终得分 = 向量得分 × 0.55
         + BM25 得分 × 0.30
         + 查询词覆盖率 × 0.15
```

最终只保留相关度高于阈值的候选资料。

当前实现是轻量本地重排，后续可以替换为 BGE Reranker 或 Cross-Encoder。

### Chroma Vector Database

用于保存文档分片及其向量，并根据用户问题进行语义检索。

### Gemini

项目当前使用：

- `gemini-2.5-flash`：对话回答、RAG 总结和报告生成。
- `gemini-embedding-001`：知识文档向量化。

### 文档处理

- `PyPDFLoader`：加载 PDF。
- `TextLoader`：加载 TXT。
- `RecursiveCharacterTextSplitter`：按指定大小和重叠长度切分文档。

### SQLite Manifest

使用 SQLite 保存知识源文件和 Chroma 分片 ID 的对应关系。

Manifest 支持：

- 新文件自动建立索引。
- 文件内容变化后更新索引。
- 源文件删除后精确删除对应向量。
- 记录内容哈希、分片 ID 和索引时间。

相比单纯保存 MD5，Manifest 能完整管理文档索引生命周期。

### CSV 数据服务

使用 `csv.DictReader` 解析带引号和多行字段的用户使用记录，并转换为结构化 `UsageRecord`。

### YAML 配置

使用 YAML 管理：

- Chroma 数据库路径。
- 文档分片参数。
- 召回数量。
- 相关度阈值。
- 混合检索权重。
- Prompt 文件路径。
- 演示用户上下文。

### 日志和 Middleware

记录：

- Agent 调用模型前的状态。
- 调用了什么工具。
- 工具输入参数。
- 工具是否成功。
- 知识库同步异常。
- 向量检索降级情况。

### 自动化测试

使用 Python `unittest` 和 Fake Model/Fake Vector Store 编写离线测试，不消耗模型 API 配额。

测试覆盖：

- Manifest 新增、更新和删除。
- 源文档增删改同步。
- 混合检索和重排顺序。
- 来源引用。
- LangGraph 报告节点执行顺序。

## 7. 项目技术亮点

### Agent 与确定性工作流结合

普通开放问题使用 Agent 保持灵活性；报告等强业务流程使用 LangGraph，兼顾自主决策能力和流程可靠性。

### 混合 RAG

不是只依赖向量检索，而是结合 BM25、向量召回和重排，提高中文产品术语、故障词及语义问题的检索效果。

### 可追溯回答

回答包含 `[来源N]`、文件名、PDF 页码和相关度，降低无依据回答带来的可信度问题。

### 检索降级机制

向量库为空或向量查询异常时，系统可以降级为本地 BM25 检索。

### 完整索引生命周期

通过 SQLite Manifest 管理文件和向量分片映射，支持知识文件的新增、修改与删除，而不只是重复文件检测。

### 可测试设计

将记录查询、上下文、检索、报告工作流拆成独立模块，可以使用 Fake 依赖进行离线测试。

## 11. 面试时可能被追问的问题

### 为什么同时使用 Agent 和 LangGraph？

开放式问答所需工具不固定，适合 Agent 自主选择；报告生成步骤固定且依赖顺序明确，使用 LangGraph 更容易控制流程、测试分支和避免漏步骤。

### 为什么不能只用向量检索？

向量检索擅长语义相似，但对型号、故障码和明确关键词不一定稳定。BM25 能补足精确词匹配能力，混合召回通常比单一路径更稳。

### Reranker 做了什么？

它对向量检索和 BM25 返回的候选重新评分，将语义相似度、关键词相关度和查询词覆盖率加权，过滤低于阈值的文档后再提供给模型。

### 如何防止 RAG 幻觉？

- 使用相关度阈值过滤不相关文档。
- Prompt 要求回答只能基于检索资料。
- 输出来源编号、文件名和页码。
- 无合格资料时明确返回未找到，而不是强制回答。

### Manifest 相比 MD5 文件有什么优势？

MD5 文件只能表示某个内容是否处理过；Manifest 同时记录文件路径、内容哈希和所有 Chroma 分片 ID，因此可以在文件修改或删除时准确更新对应向量。

### 当前项目有哪些不足？

- 天气工具目前是模拟数据。
- 会话上下文保存在内存中，服务重启后不会持久化。
- Reranker 是轻量算法，还可以升级为 Cross-Encoder。
- 尚未建立大规模 RAG 评估集。
- Streamlit 前端和 Agent 服务仍在同一进程中。

主动说明这些限制通常比声称项目已经完全生产化更可信。
